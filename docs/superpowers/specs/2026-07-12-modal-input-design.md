# 出入库/盘点录入 — 弹窗式输入

**日期**：2026-07-12
**分支**：`aux-unit`
**目标**：将出库/入库/盘点 session 页面的内联输入改为卡片点击弹窗输入，保持提交流程不变。

---

## 背景

当前三个 session 页面（`outbound_session.html` / `restock_session.html` / `stocktake_session.html`）在每个品项卡片上直接显示 input + segment 单位切换。卡片内容拥挤，误触率高，移动端体验差。

## 设计

### 卡片布局（第一层）

每张 `.entry-row` 卡片只显示品项信息，无输入框：

```
┌─────────────────────────────────┐
│ 包材                   已填 24 个│  ← 已填时淡绿背景 + 摘要
│ 奶茶                          │
│ 当前 10 箱   安全 2 箱         │
└─────────────────────────────────┘
```

- 未填：白色背景 (`background: transparent` 或 `#fff`)
- 已填：淡绿色背景 (`#ecfdf5`)，右上角显示实际输入值摘要（如「已填 24 个」或「已填 2 箱」）
- 整张卡片可点击 (`cursor: pointer`)

### 弹窗布局（第二层）

点击卡片弹出居中浮层，遮罩层覆盖全屏：

```
┌───────────────────────────────────┐
│ 奶茶                    ✕ 关闭    │
│ 当前库存: 10 箱                   │
│                                   │
│ 单位选择                          │
│ [ 箱 ]      [ 个 ]    ← 分开 pill │
│                                   │
│ 数量                              │
│ [      24        ]                │
│ 24 个 ≈ 2 箱      ← 换算提示     │
│                                   │
│ [ 确认 ]          [ 取消 ]        │
└───────────────────────────────────┘
```

- 弹窗固定宽度（桌面 360px，移动端自适应 90vw）
- 居中定位，`position: fixed; inset: 0; margin: auto`
- 半透明遮罩层

### 单位选择 pill

- 两个独立的圆形 pill 按钮，间距 12px
- 选中的 pill：品牌色实心填充，白色文字
- 未选中的 pill：灰色边框，灰色文字
- 辅单位仅在该品项配置了 `aux_unit` / `aux_rate > 0` 时显示

### 数据流

1. 用户在弹窗内填写数量、选择单位、点击「确认」
2. 确认时写入卡片对应的 `<input type="hidden">` 字段（`outbound_{{id}}` + `outbound_{{id}}_unit`）
3. 卡片背景变淡绿，右上角显示摘要「已填 24 个」或「已填 2 箱」（跟随所选单位）
4. 再次点击可修改
5. 点击「提交并出库」一次性 POST 提交全部（后端零改动）

### 移动端

- 弹窗居中悬浮（非全屏），`max-width: 90vw`
- 输入法弹出时弹窗位置不变

---

## 风险

- **精度一致性**：弹窗内换算提示用 JS 浮点运算（`Math.round(x*100)/100`），与后端 Decimal 结果可能差 0.01，仅展示用，不影响入库数据
- **多次点击**：已填卡片的弹窗预填上次值，避免重复输入
